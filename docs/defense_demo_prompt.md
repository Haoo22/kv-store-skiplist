# KV-Store Graduation Defense Demo Prompt

设计一个适用于毕业答辩现场展示的本地可视化 demo 页面，服务对象是一个基于 C++、epoll、跳表和 WAL 的 KV-Store 项目。页面必须传达“这是一个真实做过实现、协议验证、性能分析和优化取舍的系统”，而不是抽象概念展示。

## 目标

- 页面第一眼要有“系统正在运行”的感觉
- 适合投屏展示，布局清晰，视觉节奏强
- 同时体现协议交互、并发客户端、性能统计和结论摘要
- 不依赖外部服务，适合本地离线演示

## 核心展示内容

1. 多个终端窗格，模拟多个客户端同时向 KV-Store 发送命令和接收响应
2. 一个中心状态区，展示当前演示模式、连接数、QPS、延迟、key 数量、WAL 状态
3. 一个动态图表区，展示 aggregate QPS、phase 吞吐和各客户端活跃度
4. 一个结论区，用极简语言概括：
   - 单线程 Reactor 是当前主线版本
   - 支持文本协议 `PING/PUT/GET/DEL/SCAN/QUIT`
   - 已做过 benchmark 与线程池反证实验
   - 线程池版本端到端退化，因此保留单线程基线

## 视觉方向

- 风格：偏工程演示舱、终端指挥台、实验台，而不是普通后台
- 背景：深色基底叠加渐变光斑和网格纹理
- 主色：青绿色、琥珀色、红橙色，避免紫色
- 字体：终端区使用等宽字体，标题和指标使用更有展示感的非默认字体风格
- 动画：轻微呼吸光、终端滚动、柱状图增长、折线图推进

## 布局要求

- 顶部：项目标题、模式按钮、一键开始演示按钮、重置按钮
- 左侧：多客户端终端矩阵，客户端数量可以动态变化，但默认展示 6-10 个
- 右侧：关键指标卡片和系统结构摘要
- 底部：动态图表，包括：
  - aggregate QPS 柱状图
  - 延迟趋势折线图
  - 每客户端请求进度条

## 交互要求

- 有至少三种演示模式：
  - Protocol Walkthrough
  - Concurrent Pressure
  - WAL Recovery
- 点击开始演示后，终端窗格应逐步滚动输出命令与响应，而不是瞬间全部出现
- 图表和指标必须随演示节奏动态变化
- 重置后回到初始状态

## 内容细节

- 终端输出要贴近真实协议，例如：
  - `PING`
  - `PUT user alice`
  - `GET user`
  - `SCAN a z`
  - `DEL user`
  - `QUIT`
- 响应要贴近真实项目：
  - `PONG`
  - `OK PUT`
  - `OK UPDATE`
  - `VALUE alice`
  - `RESULT 1 user=alice`
  - `OK DELETE`
  - `BYE`
- 并发压力模式下，各客户端使用不同 key 前缀，体现并行压力而不是重复同一条命令

## 工程约束

- 只用原生 HTML/CSS/JavaScript
- 不依赖 npm、CDN 或构建工具
- 页面单文件或少量静态文件即可运行
- 代码结构清晰，方便后续继续接真实 benchmark 输出

## 最终效果

做出来的页面应该让老师在几秒内感知到：

- 这是一个真正实现了的网络存储系统
- 它有协议、有状态、有压力测试、有实验结论
- 作者不只是“做了功能”，还做了工程化验证与性能分析
