# KV-Store 线程池实验结论摘要

本文档用于把线程池方案的实验结论从 README 长日志中提炼出来，便于后续论文写作和答辩引用。

## 1. 实验目标

原始问题是：

- 单线程 Reactor 是否限制了端到端吞吐
- 如果把命令执行交给线程池，吞吐是否会提升

## 2. 实验结论

当前仓库的结论已经很明确：

- 线程池方案在单客户端 benchmark 中显著落后于旧版单线程 Reactor
- 在多客户端 aggregate QPS 场景下，也没有表现出应有的扩展性
- 因此线程池方案不进入主线

## 3. 原因摘要

退化的主要原因可概括为：

1. I/O 入口仍在主 Reactor 线程
2. 每条命令都引入跨线程排队和唤醒成本
3. completion 回传与重排增加了额外串行点
4. 最终写回仍回到 Reactor 线程
5. `KVStore` 内部自身也有锁，线程切换未必换来真实并行度

## 4. 论文中的推荐写法

推荐把线程池方案写成：

- 一次有价值但失败的并行化探索
- 它证明了“并发结构设计必须结合端到端路径评估”
- 最终主线保留单线程 Reactor，是基于实验结果而不是保守假设

## 5. 当前定位

- 线程池方案：实验反例
- 单线程 Reactor：当前主线
- benchmark：裁决依据
